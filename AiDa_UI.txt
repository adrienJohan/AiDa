# AiDa UI & UX Specification (v1)

## Product Vision

AiDa is not a chatbot.

AiDa is a personal fitness and nutrition workspace powered by an AI coach.

The user should feel like they are working with a supportive coach that understands their goals and follows their progress over time.

The UI should feel:

* Modern
* Clean
* Calm
* Professional
* Human

The UI should NOT feel:

* Like ChatGPT
* Like Discord
* Like Messenger
* Like a generic student project

---

# Design Principles

## Principle 1

AiDa talks.

Pages visualize.

The AI is responsible for:

* Coaching
* Guidance
* Explanations
* Recommendations
* Motivation

Pages are responsible for:

* Plans
* Reports
* Progress
* History
* Visualization

---

## Principle 2

Conversation generates objects.

Example:

Conversation
→ Workout Plan Card

Conversation
→ Meal Plan Card

Conversation
→ Weekly Report Card

Conversation
→ Progress Update

The application should avoid large walls of generated text.

---

## Principle 3

Always tell the user what to do next.

The application should never feel empty.

Whenever possible, AiDa should proactively suggest the next action.

# Landing Page

Purpose:

Create trust and encourage the user to begin.

Contains:

AiDa Logo

Main Headline

Short Description

Start Your Journey Button

No analytics.

No dashboard.

No profile information.

---

# Onboarding Conversation

Purpose:

Collect profile information naturally.

Feels like chatting with a coach.

The user should never see forms.

AiDa asks questions conversationally.

Examples:

Tell me about yourself and your fitness goals.

What equipment do you have access to?

Do you have any injuries I should know about?

---

# Profile Complete Screen

Purpose:

Mark the end of setup.

Example:

Profile Complete

Welcome Adrien

Ready to begin your fitness journey?

Button:

Enter AiDa

---

# Mission Control

Purpose:

Answer:

"What should I do next?"

Mission Control is NOT a dashboard.

Mission Control evolves depending on user state.

---

Create Workout Plan

---

## Active User State

Weight Change

Recommended Action

# Coach Page

Purpose:

Main interaction area.

The most frequently used page.

AiDa always starts the conversation.

The page should never appear empty.

Examples:

Welcome back Adrien.

You completed one workout since our last conversation.

What would you like to work on today?

---

# AiDa Personality

Supportive

Realistic

Action-Oriented

Examples:

Good:

"Nice work. That's another step toward your goal."

Good:

"Progress has been slower this week. Improving workout consistency could help."

Avoid:

Excessive hype

Excessive emojis

Unrealistic praise

Generic motivational quotes
